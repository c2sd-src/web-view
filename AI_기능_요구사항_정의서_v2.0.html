Overview

왜 EAI/ESB에 AI를 내장하는가

25년간 150개 고객사에 정보연계 서비스를 제공해 온 모코엠시스의 구조적 한계를 AI로 돌파합니다.

PROBLEM

인력 의존적 매출 구조

솔루션 비즈니스사업부 64명이 고객 Site에 3~12개월 상주하며 Customizing. S/W 매출이 인원 가용성에 직접 연동되어 영업이익률이 낮음.

PROBLEM

제품 노후화 20년

2000년대 중반 이후 제품 투자 미비. Cloud Native, API-First, Low Code 등 최신 트렌드 미대응. 과거 Renewal 시도 실패 경험 축적.

VISION

Passive → Active Intelligence

'수동적 연결(Passive Connection)'에서 '능동적 지능화 연결(Active Intelligence Connection)'로의 패러다임 전환. 궁극적으로 Low Code/No Code 정보연계 도구 완성.

Classification

7개 분류체계, 23건 요구사항

기반 인프라부터 고객가치 확장까지, 기술 의존관계를 반영한 분류입니다.

3
A

기반 인프라

LLM 추상화 레이어, RAG 파이프라인, 시스템 메타데이터 수집 프레임워크. 모든 AI 기능의 공통 토대.

5
B

지능형 설계/개발

자연어 기반 인터페이스 정의, 시맨틱 매핑, 코드 자동생성, 변환 룰, 플로우 디자이너. M/M 절감의 핵심.

2
C

인터페이스 디스커버리

메타 기반 인터페이스 자동 도출/추천(★W-11) + 라이브 시연 엔진(★W-01). 킬러 데모의 핵심.

2
D

테스트 자동화

에지케이스 시나리오 자동설계, 시맨틱 가상 데이터 생성. 배포 전 리스크 최소화.

5
E

자율 운영

장애 자동진단, 헬스 스코어(★W-02), 자가 치유(★W-03), ChatOps(★W-09), 예측 경고.

3
F

보안/거버넌스

PII 실시간 마스킹(★W-04), 규제 준수 자동검사(★W-10), AI 에이전트 권한/쿼터 관리.

6
G

고객가치/확장

에이전트 허브 MCP(★W-05), 데이터 큐레이션(★W-06), 비즈니스 영향도(★W-07), 마이그레이션(★W-08), 문서/제안서.

Roadmap

5-Phase 기술검증 로드맵

기술 의존관계에 따라 기반→설계→디스커버리→운영→확장 순서로 10개월간 검증합니다.
